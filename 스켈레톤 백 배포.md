### 🚀 전략 1: 숏 폴링 (Short Polling) - "눈치 못 채게 계속 물어보기" (강력 추천)

가장 현실적이고 프론트 단에서 5분이면 구현할 수 있는 방법입니다. Vue 컴포넌트가 Glitch 서버에 "혹시 순위 바뀐 거 있어?" 하고 주기적으로(예: 3초마다) 계속 요청을 보내는 겁니다.

**구현 방법 (Vue 3 Composition API 기준):** 챌린지 랭킹을 보여주는 컴포넌트(`ChallengeView.vue` 등)에 `setInterval`을 걸어줍니다.

```js
<script setup>
import { ref, onMounted, onUnmounted } from 'vue'
import axios from 'axios'

const rankingList = ref([])
let pollingTimer = null // 타이머 변수

// 랭킹 데이터 불러오는 함수
const fetchRanking = async () => {
  try {
    // Glitch 서버로 요청 (예: 잔액 기준으로 내림차순 정렬)
    const response = await axios.get('https://여러분의-glitch-주소.glitch.me/users?_sort=-remainingBudget')
    rankingList.value = response.data
  } catch (error) {
    console.error('랭킹 못 가져옴:', error)
  }
}

onMounted(() => {
  fetchRanking() // 처음 들어왔을 때 1번 호출
  
  // 3초(3000ms)마다 백그라운드에서 조용히 랭킹 갱신
  pollingTimer = setInterval(() => {
    fetchRanking()
  }, 3000) 
})

onUnmounted(() => {
  // 컴포넌트 나갈 때 타이머 안 끄면 메모리 누수 발생! (전공자의 디테일)
  clearInterval(pollingTimer)
})
</script>
```

- **효과:** 발표 때 A 팀원이 자기 폰으로 커피값 5,000원을 지출(POST)하면, B 팀원이 띄워둔 랭킹 화면이 3초 뒤에 스르륵 하고 새로고침 없이 순위가 바뀝니다. 완벽한 실시간처럼 보이죠!

### 🚀 백엔드 선(先) 배포 & 로컬 프론트 연동 3단계

**1. Glitch에 json-server 배포하기**

- Github에 `db.json`, `package.json`, `server.js` (json-server 구동 스크립트)가 포함된 레포지토리를 만듭니다.
    
- Glitch에서 **'Import from GitHub'**를 통해 해당 레포지토리를 불러와서 서버를 띄웁니다.
    
- 배포가 완료되면 Glitch 프로젝트 주소(예: `https://yageum-api.glitch.me`)를 복사해 둡니다.
    

**2. 프론트엔드 `.env` 파일 수정 (로컬 작업)**

- 프론트엔드 팀원들에게 "이제 백엔드 API 주소 나왔으니까 환경 변수 수정하세요!"라고 전파합니다.
    
- 프론트엔드 프로젝트 최상단 `.env` 파일의 API 주소를 방금 만든 Glitch 주소로 바꿉니다.
    ```Plaintext
    VITE_API_BASE_URL=https://yageum-api.glitch.me
    ```
    

**3. 로컬에서 프론트 서버 띄우고 테스트**

- 프론트엔드 터미널에서 `npm run dev`를 쳐서 `localhost:5173`으로 접속합니다.
    
- 화면에서 지출을 추가해 보고, 네트워크 탭이나 Glitch 로그를 확인해서 외부 서버로 데이터가 잘 넘어가는지(POST) 확인합니다.
    

이렇게 세팅해 두면 프론트엔드 팀원들은 Vercel 배포 신경 쓸 필요 없이, 각자 로컬 컴퓨터에서 편하게 띄워놓고 빵빵한 Glitch 백엔드 데이터를 바라보며 주말 내내 화면만 예쁘게 다듬으면 됩니다. 백엔드 배포부터 빠르게 밀어붙여 보시죠!
